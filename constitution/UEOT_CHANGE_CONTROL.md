# UEOT Change Control

## Change Classes

### Class A: Constitutional
Changes to core postulates, canonical definitions, or method doctrine.

Required:

- RFC
- cross-domain impact analysis
- architect resolution

### Class B: Domain-Structural
Changes to a domain's accepted claims, frontier map, or standard diagnostics.

Required:

- domain memo or cross-review
- architect resolution or acceptance marker in domain state

### Class C: Book-Assembly
Changes to chapter structure, target theses, or chapter dependency graphs.

Required:

- architect approval
- updated chapter dossier

## RFC Template

- RFC id
- authoring agent
- scope
- current rule
- proposed rule
- reason for change
- evidence basis
- affected domains
- expected monograph impact

## Invalid Changes

The following are invalid and should be reverted or ignored:

- silent rewrites of core terms
- direct edits to another domain's accepted claims without review
- canon changes without architect resolution
- book claims with no memo lineage
