# UEOT Constitution

## Purpose

This document defines the non-negotiable constitutional core of UEOT. All
memos, reviews, resolutions, and book chapters must remain consistent with this
layer unless a formal RFC and resolution explicitly changes it.

## Foundational Commitments

1. **Objecthood is processual, not merely substantial.**
   A UEOT object is a persistently existing entity whose identity is maintained
   by at least one self-regenerating internal feedback structure.

2. **The persistence condition is topological.**
   The minimum ontological condition for persistent existence is the presence of
   an `Omega`-loop, understood as a self-maintaining loop that regenerates the
   object's pattern over time.

3. **Evolution is governed by coupled optimization.**
   Every object evolves under the coupled action of:
   - `Pi`: physical efficiency, payoff, economy, viability, or energetic gain
   - `Phi`: informational ordering, uncertainty reduction, learning, or
     complexity-preserving adaptation

4. **Teleology is emergent, not imported from outside nature.**
   UEOT interprets goal-directedness as the macro-scale expression of an
   object's internal dynamical optimization landscape.

5. **Every serious domain extension must express GOD and GOA.**
   - `GOD` is the local direction of evolution.
   - `GOA` is the longer-run attractor or optimal state toward which the object
     tends.

6. **Lifecycle logic is mandatory.**
   Every object analysis must specify the object's lifecycle in terms of:
   - emergence
   - maturation / optimization
   - decay / dissolution

7. **Multiscale analysis is mandatory where applicable.**
   Agents must consider nested objects, sub-object relations, super-object
   boundary conditions, and RG-style scale transitions when the domain requires
   them.

8. **UEOT remains a research program, not a completed final theory.**
   Missing equations, unresolved conjectures, and incomplete evidence must be
   stated explicitly.

## Constitutional Invariants

The following are constitutional invariants and may not be overwritten by
domain agents:

- persistent existence is grounded in `Omega`-loop structure
- evolution is driven by `Pi/Phi` coupling
- purposeful behavior is interpretable via `GOD/GOA`
- object analysis requires lifecycle staging
- multiscale and RG-style reasoning are first-class UEOT commitments

## Permitted Extensions

Domain agents may:

- restate the core in domain-specific language
- propose domain-specific definitions of objects and loops
- formulate domain-specific `Pi/Phi` tensions
- propose mathematical formalizations
- define diagnostics, experiments, and falsification paths
- identify tensions with external theories

Domain agents may not:

- rewrite the meaning of `Omega`-loop, `Pi`, `Phi`, `GOD`, or `GOA`
- declare conjectural claims canonical without resolution
- erase unresolved objections
- silently replace UEOT language with another framework

## Constitutional Change Rule

Any modification to this file requires:

1. a written RFC
2. explicit statement of the old and proposed rule
3. rationale and evidence basis
4. cross-domain impact analysis
5. an architect resolution stored in `council/resolutions/`

Without that flow, constitutional edits are invalid.
