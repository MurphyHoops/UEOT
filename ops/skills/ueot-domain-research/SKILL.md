# UEOT Domain Research

Use this skill for domain-specific UEOT work.

## Workflow

1. Load `memory/shared/UEOT_MEMORY_REGISTRY.yaml`.
2. Load the constitution.
3. Read your domain `STATE.md`.
4. Read the most recent accepted memo in your domain.
5. Read any directly relevant council resolutions and cross-domain reviews.
6. Produce one memo or memo revision.

## Required Memo Elements

- domain object framing
- candidate `Omega`-loops
- `Pi/Phi` translation
- `GOD/GOA`
- lifecycle staging
- external literature stress test
- at least 2 cross-domain links
- at least 1 falsifiable or actionable consequence
- explicit references to the shared registry when citing accepted dependencies
