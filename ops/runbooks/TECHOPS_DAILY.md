# TechOps Daily

1. Check OpenClaw agent health.
2. Check cron health and recent runs.
3. Check memory indexing state.
4. Check whether any task would benefit from a built-in or external skill.
5. If no suitable skill exists, create or refine a local UEOT skill.
6. Record operational issues in shared memory and runbooks.

Constraints:

- do not change theory content
- prefer deterministic scripts over repeated manual steps
- treat missing automation as TechOps debt
