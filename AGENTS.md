# UEOT Omega Repo Instructions

This repository is the canonical UEOT research hub for the OpenClaw multi-agent
system.

Before doing substantive work:

1. Read [constitution/UEOT_CONSTITUTION.md](constitution/UEOT_CONSTITUTION.md).
2. Read [constitution/UEOT_METHOD.md](constitution/UEOT_METHOD.md).
3. Read the relevant [domains](domains) `STATE.md`.
4. Read the relevant [book](book/chapters) dossier if your work targets the book.

Operating rules:

- Treat the original paper as the constitutional baseline.
- Extend UEOT through explicit written claims, not silent reinterpretation.
- Domain agents may only write inside their own domain directories, shared
  memory, council review notes, or assigned book dossiers.
- Changes to core postulates, canonical definitions, or global method require a
  written architect resolution first.
- Every accepted research output must show:
  - at least one vertical link to the constitutional core
  - at least one horizontal link to another domain
  - at least one falsifiable or research-actionable consequence

File contracts:

- Research memos live in `domains/<domain>/memos/`.
- Cross-domain critiques live in `council/reviews/`.
- Architect rulings live in `council/resolutions/`.
- Book work lives in `book/chapters/`.

Evidence policy:

- Use the original UEOT paper first.
- Use external literature for support, contrast, stress testing, and possible
  falsification.
- Do not present external theories as if they are already part of UEOT canon.

Authoring policy:

- Canonical theory documents and accepted research outputs are written in
  English.
- Chinese is allowed in operational notes and human-facing summaries outside
  canonical research documents.
